TL;DR (cập nhật)

	•	Android Security Bulletin (ASB) 2025-08 đã phát hành; patch level 2025-08-05 là baseline để “đủ” toàn bộ lỗ hổng, trong đó có RCE ở System.  ￼

	•	Pixel Update Bulletin 2025-08: mọi máy Pixel hỗ trợ sẽ nhận 2025-08-05, có cả bản vá modem.  ￼

	•	Google Play policy: từ 2025-08-31, app mới & cập nhật phải target API 35 (Android 15); có khoảng gia hạn đến 2025-11-01. Wear/Auto/TV vẫn API 34.  ￼ ￼

	•	Chrome/WebView 139: ngừng hỗ trợ Android 8/9, yêu cầu Android 10+; WebView dùng chung engine Chromium với Chrome ⇒ thiết bị 8/9 sẽ “đóng băng” engine.  ￼ ￼ ￼

	•	iOS/iPadOS 18.6 (29/07/2025): nhiều mục WebKit (ví dụ CVE-2025-43216/43228/43265/6558) ảnh hưởng trực tiếp WKWebView trong app.  ￼

	•	Frida 17.2.15: hỗ trợ iOS 26 (dual-mapping, injection trên palera1n), cải thiện frida-java-bridge phía Android; cần cập nhật heuristic RASP.  ￼

Điểm nhấn (đã làm rõ)



\[P0] android/security_patch — ASB 2025-08

	•	Tóm tắt kỹ thuật: Bulletin 8/2025 yêu cầu 2025-08-05 để bao phủ tất cả vấn đề; có RCE (Critical) ở System “zero-click” khi kết hợp với bug khác.  ￼

	•	Ảnh hưởng tới banking:

	•	Bề mặt web/media/system có thể dẫn tới crash hoặc RCE → tác động đăng nhập, OTP, giao dịch, KYC camera, và in-app web.

	•	Tùy SoC còn có Arm/Qualcomm/closed-source ở nhánh 08-05.  ￼

	•	Hành động:

	1.	Baseline QA ≥ 2025-08-05 trên ma trận thiết bị.

	2.	Smoke: đăng nhập → OTP → giao dịch; KYC camera; tất cả màn có WebView.

	3.	Theo dõi crash/ANR theo OS/WebView version; cảnh báo spike bất thường.



\[P1] android/store_policy — Target API 35 từ 31/08/2025

	•	Quy định: Mọi bản cập nhật & app mới phải target 35; Wear/Auto/TV 34. Có đơn xin gia hạn đến 01/11/2025.  ￼ ￼

	•	Ảnh hưởng:

	•	Cần rà soát scope quyền (Health/Body Sensor, notification, media permissions, background).

	•	App đang target 34 vẫn phân phối được, nhưng update tiếp theo phải nâng 35.

	•	Hành động:

	•	Nâng targetSdkVersion=35; cập nhật AGP/Gradle; chạy SDK Upgrade Assistant.

	•	Thiết lập regression matrix API 29 → 35 (đặc biệt: scoped storage/foreground service/notifications).



\[P1] ios/security_patch — iOS/iPadOS 18.6

	•	Tóm tắt: Phát hành 29/07/2025; có nhiều mục WebKit liên quan điều khiển thanh địa chỉ, OOB read, UAF crash (ví dụ CVE-2025-43228/43265/43216/6558).  ￼

	•	Ảnh hưởng tới banking: rủi ro SSO/3DS/statement viewer/deep link chạy trong WKWebView.

	•	Hành động:

	1.	Baseline QA iOS/iPadOS 18.6; regression hành vi AutoFill OTP & 3DS.

	2.	Kiểm tra ATS/pinning và UI anti-spoofing sau khi engine WebKit thay đổi.

	3.	Theo dõi RSR và đối chiếu WebKitGTK/WPE để ước lượng phạm vi cross-platform.  ￼



\[P2] android/sdk_tooling — Chrome/WebView 139 ngừng Android 8/9

	•	Thay đổi: Chrome 138 là bản cuối cho Android 8/9; Chrome 139 (5/8/2025) yêu cầu Android 10+. Vì WebView dùng chung engine với Chrome, thiết bị 8/9 ngưng nhận cập nhật engine.  ￼ ￼ ￼

	•	Ảnh hưởng: khác biệt rendering/JS, rủi ro CVE web; ảnh hưởng trực tiếp 3DS, statement viewer, OAuth/SSO in-app.

	•	Hành động:

	•	Nâng yêu cầu Android 10+ cho luồng web nhạy cảm.

	•	Thiết lập guardrail (ví dụ: read-only, cấm thanh toán) nếu WebView quá cũ; hiển thị banner nâng OS.

	•	Quan sát tỉ lệ 8/9 và lên lộ trình khuyến khích nâng cấp thiết bị.



\[P2] cross-platform/security_patch — WebKitGTK/WPE WSA-2025-0005

	•	Ý nghĩa: advisory tổng hợp các CVE WebKit tương đồng với iOS 18.6 (ví dụ CVE-2025-43228/43265/6558), tiện để map phạm vi trên Linux/embedded & so với WKWebView.  ￼



\[P2] root_jailbreak/hook_tamper — Frida 17.2.15

	•	Thay đổi chính: hỗ trợ iOS 26 (dual-mapping, injection trên palera1n); Android bump frida-java-bridge với cải thiện lookup offset trên ART; thêm visionOS; các chỉnh sửa register/NativeFunction.  ￼

	•	Tác động RASP: fingerprint “cũ” theo process/lib/pipe/port có thể bỏ lọt; cần heuristic dựa vào hành vi (IPC, injection) và điểm kiểm soát giao dịch.

Android (chi tiết triển khai)



Android Security Bulletin — August 2025



Rủi ro: RCE ở System (Critical) + EoP trong Framework, và vendor patches (Arm/Qualcomm/closed-source) ở mức 08-05. Patch level 2025-08-05 bao phủ toàn bộ.  ￼



Checklist QA

	•	Thiết bị test: tối thiểu Pixel (patch 08-05) + 1–2 OEM phổ biến.  ￼

	•	Kịch bản smoke:

	•	Đăng nhập → OTP (SMS/Call/Email) → giao dịch.

	•	KYC camera (permission/capture/encode/upload).

	•	In-app web: 3DS, statement/PDF, OAuth/SSO.

	•	Theo dõi sau phát hành: crash/ANR theo OS patch & WebView version.



Pixel Update Bulletin — August 2025



Lưu ý modem/telephony: có RCE High ở modem (CVE-2025-26784) trong mục Pixel; yêu cầu 08-05 cho mọi máy được hỗ trợ. Kiểm tra SMS OTP/voice call và push sau update.  ￼



⸻



Google Play Policy — Target API 35 (từ 31/08/2025)



Yêu cầu: app mới & cập nhật phải target API 35; Wear/Auto/TV target 34; có extension đến 01/11/2025.  ￼ ￼



Checklist kỹ thuật

	•	compileSdk=35, targetSdk=35, AGP ≥ phiên bản tương thích.

	•	Chạy SDK Upgrade Assistant; phủ regression API 29→35 (storage/notifications/bg-service).

	•	Rà soát disclosure cho các quyền nhạy cảm (Health/Body Sensor…).



⸻



Chrome/WebView 139 — kết thúc hỗ trợ Android 8/9



Thay đổi nền tảng: Chrome 139 yêu cầu Android 10+; 138 là bản cuối cho Android 8/9. Vì WebView share engine với Chrome, thiết bị 8/9 sẽ không còn cập nhật engine (gồm vá bảo mật & thay đổi rendering).  ￼ ￼ ￼



Hành động đề xuất

	•	Test matrix: yêu cầu Android 10+ cho mọi luồng web nhạy cảm (3DS/SSO/statement).

	•	Guardrails khi WebView quá cũ:

	•	Soft block thanh toán / bắt buộc mở Chrome Custom Tabs nếu khả dụng.

	•	Banner khuyến nghị nâng OS.

	•	Theo dõi: tỷ lệ người dùng 8/9 theo release channel; lộ trình khuyến khích upgrade.



⸻



iOS / iPadOS 18.6 — Security Content



Ngày phát hành: 29/07/2025.

WebKit: nhiều mục liên quan address bar spoofing, OOB read, crash (UAF); ví dụ CVE-2025-43228/43265/43216/6558. Ảnh hưởng trực tiếp WKWebView (SSO/3DS/viewer).  ￼



Checklist QA

	•	Baseline iOS/iPadOS 18.6 cho thiết bị test.

	•	Regression: login, OTP AutoFill, 3DS, statement/PDF, deep link.

	•	Kiểm tra ATS/pinning và UI anti-spoofing khi WebKit đổi.

	•	Theo dõi RSR & đối chiếu với WebKitGTK/WPE WSA-2025-0005.  ￼



⸻



Root / Jailbreak / Hook — Frida 17.2.15



Thay đổi đáng chú ý:

	•	iOS 26: hỗ trợ thông qua dual-mapping, cho phép injection (palera1n).

	•	Android: nâng frida-java-bridge (cải thiện lookup offset trên ART).

	•	Khác: visionOS, NEON registers ARM/arm64, cập nhật NativeFunction.  ￼



Khuyến nghị RASP

	•	Bổ sung phát hiện process injection / code-sign anomalies / IPC bất thường thay vì chỉ dựa tên lib/process/pipe.

	•	Chèn attestation (Play Integrity / App Attest) tại màn nhạy cảm & trước giao dịch.

	•	Thu thập telemetry ẩn danh để theo dõi tỉ lệ nghi vấn hook/root.

	•	Bổ sung kịch bản chaos/regression: 3DS, statement viewer, OAuth/SSO trên cả WebView/WKWebView.
